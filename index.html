Attention Please!

2023.07 - 2023.08 (7인 팀 프로젝트/백엔드 담당)

📌 Summary

발표를 두려워하는 사람들을 위한 AI기반 발표 연습 피드백 플랫폼

  • 발표자의 언어적 요소와 비언어적 요소를 분석해 피드백 제공
  • AI팀의 분석 모델과 프론트엔드 간 데이터 흐름 연결하는 Django REST API 백엔드 개발
  • Swagger 기반 API 문서화 및 Postman을 활용한 백엔드 테스트 수행
  • 발표 이력 저장 및 시각화 기능 제공으로 사용자 성장 추적 가능

🧠 Background

주변에 발표를 어려워하는 친구들이 많았습니다.
그런데 생각해보면 연습할 공간이나 발표 피드백 받을 기회가 부족하다는 것을 깨달았습니다.
그래서 연습하고 피드백까지 받을 수 있는 발표 플랫폼을 만들어보자는 데에서 이 프로젝트가 시작됐습니다.
자세나 제스처 같은 비언어적 요소까지 봐주는 기능이 있다면, 집에서도 충분히 발표 실력을 키울 수 있을 거라고 생각했습니다.

👩🏻‍💻 My Role & Contribution

  • 백엔드 전반 개발 및 API 설계/구현 담당
    Django REST Framework를 사용해 프론트엔드와 AI 모델 간의 데이터 흐름을 연결
  • Swagger 기반 API 문서 자동화
    drf-yasg를 사용하여 API 명세 자동 생성, 팀원 및 프론트와의 협업 효율성 향상
  • Postman을 활용한 API 테스트 및 시나리오 검증
    각 기능별 요청/응답 흐름 테스트 및 예외 처리 케이스 점검

🔨 Technology Stack

Django Rest Framework, Swagger, Postman

💬 회고

이 프로젝트는 제 첫 팀 프로젝트였습니다.
처음에는 백엔드와 프론트엔드가 정확히 뭐가 다른지도 잘 몰랐고, 기술적으로 모든 게 처음이었습니다. 하지만 팀원들이 배울 점이 많은 사람들이었고, 그 속에서 저도 많은 것을 배웠습니다.

저는 팀원들에게 짐이 되고 싶지 않았기 때문에, 수업이 끝난 뒤 혼자 공부하며 따라갔습니다.
덕분에 Django, Postman, Swagger 등 모든 것이 낯설었지만, 결국 백엔드 담당으로서 프로젝트를 완성할 수 있었고, 그 과정에서 큰 자신감을 얻었습니다.

이 프로젝트 이후 저는 해커톤이나 다른 프로젝트에도 적극적으로 참여하게 됐고, 그만큼 실력도 많이 성장했습니다.

기술뿐만 아니라 팀워크, 커뮤니케이션, 팀 프로젝트의 즐거움까지 배울 수 있었던 저에게 가장 뜻깊은 경험이었습니다.

🧠 Background

처음에는 아버지를 위해 엑셀 농사일기를 만들어 드렸습니다. 하지만 아버지는 컴퓨터 사용이 익숙하지 않으셔서 창이 닫히거나 문제가 생기면 저를 계속 호출해야 했습니다. 제가 멀리 있을 때는 바로 도와드릴 수 없어 답답했기 때문에, 웹 기반으로 더 직관적인 농사일기를 기획하게 되었습니다. 이번 프로젝트에서는 기획부터 디자인, 프론트엔드, 백엔드까지 전부 직접 처음으로 경험하게 되었습니다.

👩🏻‍💻 My Role & Contribution

  • Spring Boot 기반 백엔드 개발 및 전체 구조 설계
  • 공공데이터포털 기상청 API 연동으로 오늘/내일 날씨 기능 구현
  • React 기반 프론트엔드 개발 및 UI/UX 디자인 직접 진행
  • 일기 작성/조회/수정/삭제 및 엑셀 내보내기 기능 구현

🔨 Technology Stack

Spring Boot, React, PostgreSQL, OpenAPI (기상청), Docker

💬 회고

처음으로 기획부터 디자인, 백엔드와 프론트엔드까지 전 과정을 직접 담당한 프로젝트였습니다. 특히 아버지께 실제로 도움이 되는 애플리케이션을 만들 수 있었다는 점에서 뿌듯했습니다. 단순히 기술 학습을 넘어 사용자 중심 설계의 중요성을 체감했고, “누군가에게 진짜 필요한 프로그램”을 만든다는 보람을 느낄 수 있었습니다.

🧠 Background

평소 공공 수영장이나 체육시설 수강 신청 시 사이트가 마비되는 불편함을 직접 경험했고, 주변에서도 같은 문제를 호소하는 경우가 많았습니다. 이를 해결하기 위해 클라우드 기반 예약 시스템을 기획하게 되었습니다. 프로젝트 진행 도중 Kubernetes 도입이 필수화되면서, 기존 모놀리식 구조로는 의미가 없다고 판단해 MSA 전환을 결단했습니다. 이 과정에서 서비스 범위 정의, 서비스 간 통신 방식 설계, 전체 아키텍처 재개발을 단 3주 만에 마무리했습니다.

👩🏻‍💻 My Role & Contribution

  • 팀장으로서 프로젝트 기획, 일정 관리, 아키텍처 설계 총괄
  • 모놀리식 → MSA 전환 과정에서 서비스 분리 및 API 통신 구조 설계
  • JitPack 기반 공통 에러 처리 모듈 구현 및 배포
  • Presigned URL 방식 이미지 업로드 기능 구현
  • Kubernetes 기반 배포 파이프라인 설계 및 운영 자동화 기초 구축

🔨 Technology Stack

Spring Boot, Kubernetes, Docker, JitPack, AWS (EKS, S3, RDS), GitHub Actions

💬 회고

부트캠프에서 가장 큰 규모의 프로젝트였고, 팀장으로서 기획부터 개발까지 모두 이끌어야 했기에 체력적으로나 정신적으로 쉽지 않았습니다. 하지만 MSA 개념을 처음 배우고 단기간에 직접 구현해본 경험은 매우 값졌습니다. 특히 공통 모듈을 배포해 협업 효율성을 높이고, Presigned URL로 성능 최적화를 이뤄낸 점이 뿌듯했습니다. 이 프로젝트를 통해 저는 단순히 개발을 넘어 아키텍처 설계, 클라우드 환경 운영, 그리고 팀 리딩의 중요성을 체감하며 크게 성장할 수 있었습니다.
